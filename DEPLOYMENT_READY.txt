STATUS: ✅ Production Ready
ISSUE: Referral rewards stuck in pending_disbursed state when balance insufficient
FIX: Proper retry logic + admin visibility tools

================================================================================
                              WHAT WAS BROKEN
================================================================================

1. Retry Loop Bug
   - Marked pending rewards as 'pending_disbursed' 
   - But retry only looked for 'pending'
   - Result: Stuck forever when balance topped up

2. Status Tracking
   - All failures collapsed to 'pending' 
   - No way to distinguish between different failure modes
   
3. Admin Visibility
   - No way to check REFERRAL_KEY balance
   - No visibility into pending queue
   - No way to verify system health

================================================================================
                           WHAT HAS BEEN FIXED
================================================================================

✅ Code Changes:
   1. main.py - Proper status differentiation (pending vs pending_disbursed)
   2. referral_service.py - Retry now handles BOTH status values
   3. routes.py - New admin endpoint for balance checking

✅ New Admin Tools:
   1. GET /api/admin/referral/key-balance - Check balance & pending queue
   2. Updated POST /api/admin/referral/process-pending - Retries pending_disbursed
   3. Method: get_pending_disbursement_summary() - See stuck rewards

✅ Documentation:
